# Frontend Engineering Notes

This repository is a collection of frontend interview notes, implementation exercises, and quick reference material.

It is organized by topic:

- `JS/` for JavaScript fundamentals, scope, closures, hoisting, and practice implementations
- `System Design/` for frontend-oriented system design notes
- `Web Vitals/` for performance metrics such as LCP, INP, and CLS
- `assets/` for supporting images used in the notes

## Contents

### JavaScript

The `JS/notes/` folder contains concise explanations of core JavaScript concepts:

- execution context
- hoisting
- scope chain
- block scope and shadowing
- closures
- `this`
- functions
- browser event behavior

The `JS/code/` folder contains practice implementations of common interview problems and language primitives:

- `once`
- `map`
- `filter`
- `reduce`
- `bind`

The `JS/gotcha/` folder contains short notes on edge cases and pitfalls.

### System Design

The `System Design/` folder contains short frontend system design writeups, including:

- RADIO and rendering strategy notes
- rendering approaches
- other design-oriented notes

### Web Vitals

The `Web Vitals/` folder covers the main browser performance metrics:

- Largest Contentful Paint
- Interaction to Next Paint
- Cumulative Layout Shift

## How To Use

Use this repository as a personal study notebook:

1. Open a topic file directly and read the note.
2. Use the implementation files in `JS/code/` as practice prompts.
3. Revisit the Web Vitals and System Design sections when preparing for frontend interviews or performance debugging.

## Notes

- The repository is markdown-first.
- There is no build system or application runtime in the current layout.
- File names are intentionally short and topic-focused for quick navigation.
