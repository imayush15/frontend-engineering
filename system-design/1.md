# Problem

Autocomplete (search suggestions while typing in input fields like Google/Amazon). As user types, frontend fetches and shows suggestions in near real-time.

## Thinking

Start by clarifying requirements instead of assuming. Ask about fuzzy matching, number of suggestions, minimum characters, expected traffic (QPS), and whether results are personalized. This sets boundaries before designing.

## Core Issue

Every keystroke can trigger an API call → leads to high request volume, backend stress, and poor user experience (lag, flicker).

## Fix

Use debouncing to delay API calls until user pauses typing. This reduces unnecessary calls and stabilizes UI behavior.

## Edge Problem

Even with debouncing, multiple requests can be in flight. Responses may arrive out of order, causing stale suggestions to appear.

## Fix

Use AbortController to cancel previous requests before firing a new one. Ensures only the latest response updates the UI. Store controller in a persistent reference (like useRef in React).
